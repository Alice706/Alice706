兼容性问题

html

  1. table里不加tbody在ie8里整个表格就不显示
  2. 不同浏览器的默认内外边距不同
    • 设置统一的maring:0 ;padding :0

CSS

  1. ie5-8不支持opacity
    • 使用filter:alpha(opacity=60);ie5-7
  2. ie浏览器的注释文档
  3. ie8支持:first-child,但不支持:last-child。
    因为前者是css2.1标准，后者是css3标准
  4. 图片在块元素中产生空白行
    • 设置vertical-align:middle
  5. 去除链接虚线边框的问题 ： 当点击超链接后，ie6/7/8 ff会出现虚线边框 ,而opera、safari没有虚线边框
    • 解决： ie6/7中 设置为a { blr:expression(this.onFocus=this.blur()) } ie8 和 ff 都不支持expression 在ie8 、ff中设置为 :focus { outline: none; }
  6. 清除浮动用的 :after ie8以下不支持
    • .clearfix {zoom:1}

JS

  1. 阻止事件冒泡和捕获
    • ie9之前 event.canceBubble = true;
    • 阻止默认事件
      说明： preventDefault()和returnValue()前者标准写法，后者IE写法。
      解决方法：一般情况建议直接使用return false阻止，但和取消默认事件的含义是不同的。
  2. h5中操作dom元素的document.querySelector
  3. 问题说明：innerText在IE中能正常工作，但是innerText在FireFox中却不行。
    • 解决方法：在非IE浏览器中使用textContent代替innerText。
  4. offset问题
    • var scroll_top=window.pageYOffset || document.documentElement.scrollTop

  5. currentstyle和getComputedStyle兼容问题

  6. 关于使用事件 event对象，出现的兼容性问题

    document.getElementById("demo").onclick=function(e){
        var e=e||event; 
        alert(e.screenX);
    }

**浏览器兼容性相关问题及其解决思路

1、目前，各主流浏览器的新版本，对于 W3C 的标准支持很好，因此，首先保证代码符合 W3C 的标准，这是解决浏览器兼容问题的前提。

其次，对于某些支持受限的属性，针对不同的浏览器添加相应的前缀，比如-webkit-、-o-、-moz-。**

移动端

  1. 如何判断用户进来的机型
    • navigator.userAgent
  2. 如何解决手机横屏字体大小不变
    • text-size-adjust:none
