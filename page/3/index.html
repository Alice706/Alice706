如何用hexo+github创建博客

一：本地安装hexo初始化一个博客的方式

1.电脑上需要安装 node.js 和 git

2.通过git bash下载hexo

在git bash中通过 npm install hexo-cli -g下载hexo
注：如果npm比较慢，可以先安装淘宝的cnpmnpm install -g cnpm –registry=https://registry.npm.taobao.org
具体的建议先百度一下，防止这个语法已经变更了

3.hexo init blog 通过这个命令初始化一个博客

我在安装时遇到一个坑在 hexo init最后会出现一个乱码，然后提示:
npm WARN deprecated swing@1.4.2: This package is no longer maintained
这个坑是真的踩的扎心了，折腾了我很久，网上百度也是五花八门的答案，后来机缘巧合之下让我找到了答案，原来只是网络慢，一直在下载中，所以，老铁们，遇到这个问题，丢在那里，什么都不要做，过会就好了

4.以上完成以后，通过命令cd blog到blog文件夹目录下，输入：hexo server

提示： Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.
这个时候博客就搭建完成了，就这么几个步骤，我竟然花了一整天的时间才研究一个入门，泪奔不过总归是完成了，还是很有成就感的，撒花

二：markdown的使用中遇到的问题

1.源文件的位置

源文件在本地的/sources/_posts/.md中,找到这个文件，就可以在里面通过markdown语法来写博客了

2.如何发布

写好博客以后，在git bash中输入命令行：hexo g用来生成完成的博客，通过hexo s查看页面效果

3.遇到的乱码问题

在可以自己写博客时，我的心情是万分激动的，但是当我去页面查看时，发现我写的中文全部都是乱码，
乱码，乱码，重要的事情说三遍，起初以为是自己的markdown语法写错了，后来各种百度，各种查找，终于
在一篇文章中看到，原来是编码格式的问题，只要：将博客文件保存为UTF-8用记事本打开本地的博客文件“xxx.md”，然后点“另存为”，“编码(E):”选择“UTF-8”，点击“保存”，替换原文件。

4.如何创建一篇新的博客

这个只要在命令行输入：hexo new “博客名称” 就妥妥的新建了一个新的博客，你可以去源文件的位置去找到这个文件

5.关于markddown语法

语法这个东西其实不多，直接百度markdown怎么写，或者去官网看看

三：上传到github上

在github上创建一个：自己的用户名.github.io的文件
在blog（本地创建的目录）下的public下 git bash here执行命令:git status用来查看状态，git add -A用来保存全部，git commit -m “存档的名字”完成以后最后：git push 远程仓库的地址 master这里我用的是SSH的方式，可以免用户名和密码，具体的操作就不说了，想要知道可以百度

## 四：访问自己的博客
上传成功以后，在网页输入：你的用户名.github.io就可以访问自己的博客了
