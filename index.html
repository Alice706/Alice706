对象、面向对象、原型

对象

  • 无序的键值对的集合

  • 创建对象的方式

    1. 字面量：var obj = { } //缺点：大量使用时，代码的复用性太低
    2. 构造函数：var obj = new Object();//代码复用性更低
    3. 自定义构造函数：function Obj() { } var obj1 = new Obj();var obj2 = new Obj();
      • 执行过程：
      1. 创建一个对象
      2. 让this 指向这个对象
      3. 执行构造函数
      4. 返回这个对象
    4. 实例：对象，实例化：创建对象的过程
    5. 静态方法和实例方法

    • 静态方法：函数名.方法名

    • 实例方法：对象名.方法名

    • 如何给对象添加属性和方法
      1. 添加到原型上
      2. 让原型指向这个对象

  • 注：对象的返回值是引用类型时，返回的结果就是引用类型

原型

  • 概念：每个构造函数在创建出来的时候，系统会自动给这个构造函数创建并关联一个空对象，这个对象就是原型
  • 在原型中定义的方法和属性，会被这个构造函数创建出来的对象共享

面向对象

  • 面向对象就是面向过程的封装
  • 面向对象是一种编程思想
  • 原型链：对象有原型，原型本身又是一个对象，所以原型也有原型
  • 三大特性
    • 封装（可拓展性）对象对属性和方法的封装
    • 继承 （重用性）一个对象想拥有另外一个对象的属性和方法
    • 多态（可维护，可拓展）在Js中没有多态

    经典继承

  1. 简单混入 for … in(缺点：复用性低)
    var obj = {
    name : ‘nacy’;
    }
    var obj1 = {};
    for (var k in obj){
    obj1[k] = obj[k]
    }

    2. 原型混入
    - for … in 添加到原型中
    - 替换原型

    var obj = {
    name : “nacy”
    }
    function Person () {}
    var person = new Person();
    p.prototype = obj;

    3. Object.create()

    var obj = {
    name:’nacy’
    }
    var obj1 = Object.create(obj);//好用，es6的新语法，有兼容性

组件化和模块化

  • 组件化更多关注的UI部分，页面的每个部件，比如头部，内容区，弹出框甚至确认按钮都可以成为一个组件，每个组件有独立的HTML、css、js代码。

  • 可以根据需要把它放在页面的任意部位，也可以和其他组件一起形成新的组件。一个页面是各个组件的结合，可以根据需要进行组装。

  • 而模块化侧重的功能的封装，主要是针对Javascript代码，隔离、组织复制的javascript代码，将它封装成一个个具有特定功能的的模块。
    开发方式

    • 引包的方式 引入vue/dist/vue.js
    • 构建工具（webpack） 编写 xxx.vue
    • 通过引包的方式文件大小比构建工具方式的30%（因为包含了对模板编译成dom的编译器）
      vue文件
      *包含三个部分
      • 1： DOM元素
      • 2: 对当前文件所生产的东西的配置
      • 3: 样式
        vue全家桶
    • vuex能转换vue代码，成为移动端原生应用 ios android PC(大型项目必用)
    • vue-router
    • vue-resource

核心特点

  • 组件化，组件化比模块化更细分（组成页面的部件）
  • 组件：模板+动态效果+样式
  • 组件内可以使用模块，模块内不能使用组件
    双向数据流
  • 内存js对象的属性发生改变影响页面
  • 页面发生改变影响内存（js对象的属性）
    数据驱动
    • 内存的属性发生改变，驱动页面的显示做调整
      注：IE9以下不支持
      善于开发单页面应用
      热替换
  • webpack-dev-server –hot 热模块替换的好处是只替换更新的部分,而不是页面重载.
