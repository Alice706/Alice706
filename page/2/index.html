一些小的知识点

link和@import的区别

  1. link属于html标签，而@import是css提供的。
  2. 页面被加载时，link会同时被加载，而@import引用的css会等到页面加载结束后加载。
  3. link是xhtml标签，因此没有兼容性，而@import只有IE5以上才能识别。
  4. link方式样式的权重高于@import的。
  5. link支持使用js控制dom改变样式
    CDN内容分布网络
  • 避开网络上有可能影响数据传输速度和稳定性的瓶颈和环节
    return false
  • jQuery中，既能阻止事件冒泡，也能阻止默认事件
  • js中，只能阻止默认事件

      eval

    html5语义化的理解
  • 用正确的标签包含正确的内容
    cookie、sessionStorage、localeStorage的区别
  • cookie是存储在浏览器端，并且随浏览器的请求一起发送到服务器端的，它有一定的过期时间，到了过期时间自动会消失。sessionStorage和localeStorage也是存储在客户端的，同属于web Storage，比cookie的存储大小要大有8m，cookie只有4kb，localeStorage是持久化的存储在客户端，如果用户不手动清除的话，不会自动消失，会一直存在，sessionStorage也是存储在客户端，但是它的存活时间是在一个回话期间，只要浏览器的回话关闭了就会自动消失。
    重构、回流

  • 浏览器的重构指的是改变每个元素外观时所触发的浏览器行为，比如颜色，背景等样式发生了改变而进行的重新构造新外观的过程。重构不会引发页面的重新布局，不一定伴随着回流，回流指的是浏览器为了重新渲染页面的需要而进行的重新计算元素的几何大小和位置的，他的开销是非常大的，回流可以理解为渲染树需要重新进行计算，一般最好触发元素的重构，避免元素的回流；比如通过通过添加类来添加css样式，而不是直接在DOM上设置，当需要操作某一块元素时候，最好使其脱离文档流，这样就不会引起回流了，比如设置position：absolute或者fixed，或者display：none，等操作结束后在显示。

    内置对象、宿主对象、本地对象、
  • 内置对象有些对象有自己的属性和方法，无需实例化，这就是内置对象
    • 内置对象： Math Global(isNaN,parseInt(),parseFloat是它的方法)
  • 本地对象必须new实例化才能使用其属性和方法的是本地对象
    • 常见本地对象：Object Function Array Boolean Number Date RegExp

  • 宿主对象：window document body

    跨域的几种方式

  • jsonp（利用script标签的跨域能力）跨域、websocket（html5的新特性，是一种新协议）跨域、设置代理服务器（由服务器替我们向不同源的服务器请求数据）、CORS（跨源资源共享，cross origin resource sharing）、iframe跨域、postMessage(包含iframe的页面向iframe传递消息)

    http状态码

  • http状态码是表示服务器对请求的响应状态，主要分为以下几个部分

    • 1：这类响应是临时响应，只包含状态行和某些可选的响应头信息，并以空行结束

    • 2：表示请求成功，

    • 3：表示重定向

    • 4：表示客户端错误

    • 5：表示服务器端错误

    怪异模式和标准模式

    • 如果有DTD(文档声明)，就是标准模式，因为在ie6之前没有DTD
    • 标准模式是指浏览器按照W3C标准解析执行代码，怪异模式则是浏览器自己的方式解析执行代码

    web标准

    • web标准简单来说可以分为结构、表现、行为

    一次Js请求一般情况下有哪些地方会有缓存

    • DNS，CDN（内容分发网络）,浏览器，服务器

如何用hexo+github创建博客

一：本地安装hexo初始化一个博客的方式

1.电脑上需要安装 node.js 和 git

2.通过git bash下载hexo

在git bash中通过 npm install hexo-cli -g下载hexo
注：如果npm比较慢，可以先安装淘宝的cnpmnpm install -g cnpm –registry=https://registry.npm.taobao.org
具体的建议先百度一下，防止这个语法已经变更了

3.hexo init blog 通过这个命令初始化一个博客

我在安装时遇到一个坑在 hexo init最后会出现一个乱码，然后提示:
npm WARN deprecated swing@1.4.2: This package is no longer maintained
这个坑是真的踩的扎心了，折腾了我很久，网上百度也是五花八门的答案，后来机缘巧合之下让我找到了答案，原来只是网络慢，一直在下载中，所以，老铁们，遇到这个问题，丢在那里，什么都不要做，过会就好了

4.以上完成以后，通过命令cd blog到blog文件夹目录下，输入：hexo server

提示： Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.
这个时候博客就搭建完成了，就这么几个步骤，我竟然花了一整天的时间才研究一个入门，泪奔不过总归是完成了，还是很有成就感的，撒花

二：markdown的使用中遇到的问题

1.源文件的位置

源文件在本地的/sources/_posts/.md中,找到这个文件，就可以在里面通过markdown语法来写博客了

2.如何发布

写好博客以后，在git bash中输入命令行：hexo g用来生成完成的博客，通过hexo s查看页面效果

3.遇到的乱码问题

在可以自己写博客时，我的心情是万分激动的，但是当我去页面查看时，发现我写的中文全部都是乱码，
乱码，乱码，重要的事情说三遍，起初以为是自己的markdown语法写错了，后来各种百度，各种查找，终于
在一篇文章中看到，原来是编码格式的问题，只要：将博客文件保存为UTF-8用记事本打开本地的博客文件“xxx.md”，然后点“另存为”，“编码(E):”选择“UTF-8”，点击“保存”，替换原文件。

4.如何创建一篇新的博客

这个只要在命令行输入：hexo new “博客名称” 就妥妥的新建了一个新的博客，你可以去源文件的位置去找到这个文件

5.关于markddown语法

语法这个东西其实不多，直接百度markdown怎么写，或者去官网看看

三：上传到github上

在github上创建一个：自己的用户名.github.io的文件
在blog（本地创建的目录）下的public下 git bash here执行命令:git status用来查看状态，git add -A用来保存全部，git commit -m “存档的名字”完成以后最后：git push 远程仓库的地址 master这里我用的是SSH的方式，可以免用户名和密码，具体的操作就不说了，想要知道可以百度

## 四：访问自己的博客
上传成功以后，在网页输入：你的用户名.github.io就可以访问自己的博客了
