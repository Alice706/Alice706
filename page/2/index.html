require.js

前端模块化

  • 将指定功能的内容划分成一个个的模块，提高复用性，容易维护和管理

    AMD异步模块定义（Asynchronous Module Definition）

  • AMD是如何出生的
    09年由于node.js的诞生，将js语言用于服务端编程，标志了js模块化编程的正式诞生，那么有了服务端的模块，很自然的，大家就想要客户端的模块，于是AMD诞生了
  • 模块：就是实现一组特定功能的方法
  • 异步的方式加载模块，不会影响后面的语句运行，所有依赖这个模块的语句，都定义在一个回调函数中，等加载完成以后，这个回调函数才会运行
  • AMD是提前执行

require.js

  • 为什么会出现：1：管理模块之间的依赖性，便于代码的编写和维护，复用性
    2：实现js文件的异步加载，避免网页失去响应
  • 遵循的标准：AMD
  • 建议：
    如果是开发的网站含有大量的js代码，并且大多数是自己编写的js，或者开发webapp，游戏，那么推荐用requirejs；如果开发的网站含有大量第三方插件，建议放弃requirejs
  • 概念：requirejs是一个js模块的加载器，可以提高js的加载速度，避免不必要的阻塞
  • 原理：1、把所有js文件交给requirejs来管理，页面中只有一个js文件，让data-mian指向main.js文件

ajax和json以及jsonp

同源策略

  • 目的:同源政策的目的，是为了保证用户信息的安全，防止恶意的网站窃取数据。
  • 限制
    1. cookie、localStorage 和 indexDB无法读取
    2. dom 无法获取
    3. AJAX 请求不能发送
  • 规定：不同域的客户端脚本在没有明确授权的情况下，不能读写对方的资源
  • 同源：协议、域名、端口号完全一致
  • url的组成：协议 域名 端口号 路径 参数 哈希值
  • 同源策略阮一峰的博客有写
  • 链接到阮一峰的博客

AJAX(异步 Javascript 和 xml)

本质：在http协议的基础上，以异步的方式与服务器进行通信

XMLHTTPREQUEST通过在后台与服务器进行少量数据交互，实现页面异步更新
异步：某段程序执行时，不会阻塞其他程序的执行

  1. ajax是以何种格式来交互数据的？跨域的问题如何解决
    首选方案：json传输数据，jsonp实现跨域
  2. ajax解决同源策略的方案
    -JSONP
    -WebSocket
    -CORS：跨域资源分享，允许任何类型的请求
    优点：异步通信，实现页面局部刷新，按需获取数据，节约宽带资源，带来更好的用户体验
    缺点：不支持浏览器back按钮，暴露了与服务器交互的细节，对搜索引擎支持比较弱，不能很好的支持移动设备

JSON

  • JSON是一种轻量级的数据交换格式，用于在浏览器和服务器之间交换信息，本质是字符串
  • JSON和JS对象互转
    对象转换成JSON字符串：var json=JSON.stringify();
    JSON转换成对象：var obj=JSON.parse();

JSONP

  1. 与jsonp息息相关的是script标签，而xhr即ajax与之没有半毛钱关系
  2. 由于同源策略的限制，XmlHttpRequest（浏览器的内置对象，用来与后台交互信息）只允许请求当前源（域名、协议、端口）的资源，为了实现跨域请求，可以通过script标签实现跨域请求，然后在服务端输出JSON数据并执行回调函数，从而解决了跨域的数据请求，这就是jsonp的核心。
  3. jsonp是非官方跨域数据交互协议
  4. 可以跨域的标签：在浏览器中，对于标签 script、img、iframe、link 等标签都可以跨域加载资源，而不受同源策略的限制。这些带有“src”属性的标签每次加载的时候，实际上是由浏览器发起一个get请求

json和jsonp的比较

  • JSON是一种数据格式，JSONP是一种数据调用方式
  • json是理想的数据交换格式，但没办法跨域直接获取，于是就将json包裹(padding)在一个合法的js语句中作为js文件传过去。这就是json和jsonp的区别，json是想要的东西，jsonp是达到这个目的而普遍采用的一种方法，当然最终获得和处理的还是json。所以说json是目的，jsonp只是手段。json总会用到，而jsonp只有在跨域获取数据才会用到。

    ajax和jsonp的对比

  1. 相同点：都是请求一个url
  2. 不同点：AJAX是通过XMLHTTPREQUEST对象获取非页面内容（后台数据），而JSONP是动态的添加script标签来调用服务器提供的js脚本。
  3. ajax与jsonp的区别不在于是否跨域，ajax通过服务端代理一样可以实现跨域，jsonp本身也不排斥同域的数据的获取

如何用hexo+github创建博客

一：本地安装hexo初始化一个博客的方式

1.电脑上需要安装 node.js 和 git

2.通过git bash下载hexo

在git bash中通过 npm install hexo-cli -g下载hexo
注：如果npm比较慢，可以先安装淘宝的cnpmnpm install -g cnpm –registry=https://registry.npm.taobao.org
具体的建议先百度一下，防止这个语法已经变更了

3.hexo init blog 通过这个命令初始化一个博客

我在安装时遇到一个坑在 hexo init最后会出现一个乱码，然后提示:
npm WARN deprecated swing@1.4.2: This package is no longer maintained
这个坑是真的踩的扎心了，折腾了我很久，网上百度也是五花八门的答案，后来机缘巧合之下让我找到了答案，原来只是网络慢，一直在下载中，所以，老铁们，遇到这个问题，丢在那里，什么都不要做，过会就好了

4.以上完成以后，通过命令cd blog到blog文件夹目录下，输入：hexo server

提示： Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.
这个时候博客就搭建完成了，就这么几个步骤，我竟然花了一整天的时间才研究一个入门，泪奔不过总归是完成了，还是很有成就感的，撒花

二：markdown的使用中遇到的问题

1.源文件的位置

源文件在本地的/sources/_posts/.md中,找到这个文件，就可以在里面通过markdown语法来写博客了

2.如何发布

写好博客以后，在git bash中输入命令行：hexo g用来生成完成的博客，通过hexo s查看页面效果

3.遇到的乱码问题

在可以自己写博客时，我的心情是万分激动的，但是当我去页面查看时，发现我写的中文全部都是乱码，
乱码，乱码，重要的事情说三遍，起初以为是自己的markdown语法写错了，后来各种百度，各种查找，终于
在一篇文章中看到，原来是编码格式的问题，只要：将博客文件保存为UTF-8用记事本打开本地的博客文件“xxx.md”，然后点“另存为”，“编码(E):”选择“UTF-8”，点击“保存”，替换原文件。

4.如何创建一篇新的博客

这个只要在命令行输入：hexo new “博客名称” 就妥妥的新建了一个新的博客，你可以去源文件的位置去找到这个文件

5.关于markddown语法

语法这个东西其实不多，直接百度markdown怎么写，或者去官网看看

三：上传到github上

在github上创建一个：自己的用户名.github.io的文件
在blog（本地创建的目录）下的public下 git bash here执行命令:git status用来查看状态，git add -A用来保存全部，git commit -m “存档的名字”完成以后最后：git push 远程仓库的地址 master这里我用的是SSH的方式，可以免用户名和密码，具体的操作就不说了，想要知道可以百度

## 四：访问自己的博客
上传成功以后，在网页输入：你的用户名.github.io就可以访问自己的博客了
